Tradução em Linguagem de Máquina

Para que uma máquina possa executar este algoritmo o texto que escrevemos deve ser traduzido em linguagem de máquina.

Esta tradução pode ser realizada por compilação, interpretação ou por processos mistos.

Processo de Compilação

Analisadores Léxicos

Módulo Responsável pela interpretação dos Lexemas (palavras) da linguagem formal.

Lexema é um termo mais amplo por que inclui todos os conjuntos de símbolos que são possíveis em uma linguagem.

Analisador Sintático

Responsável pela análise da estrutura da linguangem.

Cabe ao analisador sintático determinar se uma determinada declaração da linguagem está corretamente formatada.

Analisador Semântico

O analisador semântico é responsável pela interpretação de cada declaração da linguagem.

Erros de tipos, declaração de variáveis, composição de argumentos, estão entre os problemas que cabem ao analisador semântico.

Geração de código intermediário

Cabe ao gerador de código intermediário, otimizar o código e transformá-lo em uma linguagem mais próxima da linguagem da máquina.

Neste módulo encontramos, atualmente, as maiores oportunidades para desenvolvimento de tecnologia. O Java, o Chrome, o LLVM são projetos que dependem do código intermediário para o seu sucesso.

Otimização de código

Este módulo pode estar dentro ou fora do gerador de código intermediário. E é responsável por fazer com que o código de máquina gerado seja o mais eficiente e eficaz possível.

Sistemas como o LLVM, possuem diversos projetos diferentes competindo pelo código mais eficiênte.

Linguagens Formais

Uma linguagem formal é uma linguagem artificial criada pelo homem, para alguma função específica.

A música é uma linguagem formal. Pelo menos, existe uma linguagem formal para a escrita, leitura e interpretação de músicas. O cálculo proposicional, que começou com Aristóteles também é uma linguagem formal. Há uma pequena diferença entre linguagens formais e regulares. Todas as regulares são formais mas, o contrário não é verdadeiro.

Linguagens Naturais

Iremos, ao longo desta disciplina, afinando, a definição de linguagem formal.

As linguagens naturais são fruto da evolução da especie humana. Um dos passos importantes nesta evolução foi o desenvolvimento de linguagens artificiais.

Alfabetos

Um alfabeto é um conjunto finito de símbolos que podem ser utilizados em uma determinada linguagem.

Nesta disciplina, vamos usar o caractere $\Sigma$ (Sigma) para representar um alfabeto com um índice para identificar sua lingagem. O $\Sigma_{bin}$ representa a linguagem binária.

Alfabetos - representação

Se desconsideramos os símbolos de pontuação, e símbolos de outros idiomas como o latim ou o grego. O alfabeto da língua portuguêsa, como falado no Brasil seria representado por:
$\Sigma_{pt_br} = {a..z,A..Z,0..9}$.

Este alfabeto, conjunto de símbolos teria a cardinalidade de 62.
Tal que $|\Sigma_{pt_br}| = |{a..z,A..Z,0..9}| = 72$.

Strings

Vamos chamar de strings a sequências finitas de símbolos de um alfabeto $\Sigma$ específico de uma linguagem.

A strings podem ter qualquer comprimento, inclusive comprimento nenhum. A esta string que existe, mas não tem comprimento daremos o nome de string vazia e representaremos pelo símbolo $\varepsilon$.

Strings

Dado o alfabeto $\Sigma={a,b}$, o conjunto de todas as strings possíveis com este alfabeto será dado por: $S={\varepsilon,a,b,ab,ba,aa,bb,aaa, ...}$

O conjunto das strings possíveis é infinito, o conjunto das strings de uma linguagem não é infinto.

Strings

Formalmente, representaremos strings como os conjuntos que são então uma string qualquer poderia ser $s={a,b,b}$, informalmente teremos $s=\text{"abb"}$.

A string $s$ tem cardinalidade 3. Contudo a $|{a,b,b}|_a = 1$.
